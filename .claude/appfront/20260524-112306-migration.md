## Naming Convention

- `sap.cloud.service`: `com.sap.myhanaapp` (added to `manifest.json` recommended)
- Naming prefix: `comsapmyhanaapp`
- appFront resource: `comsapmyhanaapp-app-front`
- appFront service-name: `comsapmyhanaapp-app-front-service`
- HTML5 module: `comsapmyhanaapp-ui`
- Deployer module: `comsapmyhanaapp-app-content`

## Auth Notes

- XSUAA retained unchanged. appFront's managed approuter uses IAS for the login flow; the XSUAA token exchange happens internally. The `redirect-uris` in `xs-security.json` cover BAS and localhost dev flows only — no CF runtime redirect URI needed.

## Known Issues / Next Steps

1. **`manifest.json` missing `sap.cloud.service`** — Consider adding `"sap.cloud": { "service": "com.sap.myhanaapp" }` to `app/interaction_items/webapp/manifest.json` for CDM/WorkZone compatibility.
2. **Build prerequisite** — `@sap/ux-ui5-tooling` (already a devDependency) provides `ui5-task-zipper`. Run `npm install` inside `appfront-migrated/app/interaction_items/` before `mbt build` to ensure the task is available.

## Build & Deploy

```bash
cd appfront-migrated
mbt build
cf deploy MyHANAApp_1.0.0.mtar
```

After deploy, verify the runtime URL with:
```bash
afctl login -a <your-appfront-api-endpoint> --sso
afctl list MyHANAApp
```
